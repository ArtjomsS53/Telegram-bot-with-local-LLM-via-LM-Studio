# Telegram Mini App для локального LLM-бота

Веб-приложение, которое открывается прямо внутри Telegram и даёт удобный интерфейс к вашему боту:
чат с моделью, управление RAG-памятью, настройки и админ-панель. Работает **полностью локально на вашем ПК**,
без хостинга и без платных сервисов.

Основной файл бота (`telegram-local-llm-bot.py`) **не менялся** — mini app это отдельный сервер,
который читает ту же базу `rag_files/rag_memory.sqlite3` и общается с тем же LM Studio.

Дизайн — премиальный «Aurora»: тёмная тема, живой анимированный градиентный фон (aurora-blobs),
glassmorphism, физика интерфейса (spring-анимации, magnetic hover, ripple), плавающая навигация
и wave-эффект свечения от строки ввода в чате.

## Экраны

- **🏠 Главная (Dashboard)** — hero с активной моделью, статистика памяти (документы, фрагменты),
  крупные плитки быстрых действий (Чат, Память, Модели, Статус, Инструменты, Загрузка).
- **💬 Чат** — стриминг ответов, выбор модели (fast / balanced / smart), сброс истории,
  премиальный ввод с glow при фокусе и живой светящейся волной. RAG-контекст подмешивается
  автоматически, показываются источники.
- **◈ Память** — карточки документов (просмотр, скачивание, удаление), живой поиск по памяти
  с процентом релевантности, и **загрузка файлов прямо из приложения** (drag-and-drop или выбор):
  PDF, DOCX, TXT, MD, LOG, CSV, JSON — текст извлекается, чанкуется и попадает в ту же RAG-память,
  что и у бота (с дедупликацией по содержимому).
- **⚡ Модели** — карточки моделей с типом, шкалой скорости, статусом «в памяти» и подсветкой активной.
- **⚙️ Инструменты** — переключатели «RAG-only» и «Режим обучения», ссылки на Статус и Админ-панель.
- **◉ Статус** — healthcheck компонентов (backend, LM Studio, RAG-база, модель эмбеддингов,
  Telegram-бот, Whisper) с цветными индикаторами.
- **🛠 Админ** (только для `ADMIN_TELEGRAM_IDS`) — статистика бота, размер/путь БД, список пользователей.

QoL: тактильный отклик (haptics), тосты, popup-подтверждения, скелетоны-загрузчики, скачивание оригиналов.

## Структура

```
miniapp/
  server.py                    # FastAPI-бэкенд (REST API + раздача фронтенда)
  static/index.html            # само мини-приложение (один файл, vanilla JS)
  requirements-miniapp.txt     # зависимости сервера
  run_miniapp.bat              # запуск сервера (Windows)
  tunnel_cloudflare.bat        # бесплатный HTTPS-туннель для Telegram
  README.md
```

## Быстрый старт (локальная проверка в браузере)

1. Убедитесь, что LM Studio запущен с включённым локальным сервером (по умолчанию `http://localhost:1234`).
2. Дважды кликните `miniapp\run_miniapp.bat` (он сам поставит зависимости и запустит сервер).
3. Откройте в браузере `http://localhost:8080`.

В браузере (без Telegram) приложение работает в dev-режиме: используется отдельный чат `dev-local`
и вы считаетесь админом — удобно всё протестировать. Первый ответ RAG-поиска может подгружать модель
эмбеддингов несколько секунд.

## Подключение к Telegram (чтобы открывалось внутри бота)

Telegram требует **HTTPS-адрес** для mini app. Локальный `http://localhost` он не примет.
Бесплатное решение без хостинга — туннель Cloudflare (аккаунт не нужен):

1. Скачайте `cloudflared.exe`:
   https://github.com/cloudflare/cloudflared/releases/latest
   (возьмите `cloudflared-windows-amd64.exe`, переименуйте в `cloudflared.exe`,
   положите в папку `miniapp`).
2. Запустите сервер: `run_miniapp.bat`.
3. Запустите туннель: `tunnel_cloudflare.bat`.
   В консоли появится адрес вида `https://something-random.trycloudflare.com`.
4. В [@BotFather](https://t.me/BotFather):
   - `/newapp` (или `/myapps` → выбрать бота) → задайте название, описание, иконку;
   - когда попросит **Web App URL**, вставьте адрес из шага 3.
   - Либо через `/setmenubutton` → выберите бота → введите тот же URL и текст кнопки (например «Открыть приложение»).
5. Откройте бота в Telegram — снизу появится кнопка меню, запускающая mini app.

> Адрес `trycloudflare.com` меняется при каждом перезапуске туннеля.
> Для постоянного адреса можно завести бесплатный именованный туннель Cloudflare
> (нужен бесплатный аккаунт и свой домен) — но для личного использования быстрый туннель проще.

Альтернатива вместо Cloudflare — [ngrok](https://ngrok.com) (бесплатный тариф):
`ngrok http 8080` даёт такой же HTTPS-адрес.

## Настройки (необязательно) через `.env`

Сервер читает тот же `.env`, что и бот. Дополнительно понимает:

```env
MINIAPP_PORT=8080            # порт сервера
MINIAPP_REQUIRE_AUTH=false   # true = запретить dev-режим из браузера (только через Telegram)
MINIAPP_DEV_CHAT_ID=dev-local
```

Модели берутся из словаря `AVAILABLE_MODELS` в `server.py` — держите его синхронным
со списком в основном боте (там же, где вы настраиваете модели LM Studio).

## Как это устроено (кратко)

- Авторизация: фронтенд передаёт `Telegram.WebApp.initData`, сервер проверяет HMAC-подпись
  секретом из `TELEGRAM_BOT_TOKEN`. Так сервер достоверно знает Telegram-ID пользователя и
  определяет права админа. Данные каждого пользователя изолированы по `chat_id`.
- Чат идёт напрямую в LM Studio (`/v1/chat/completions`, stream=true) и отдаётся на фронт
  построчным ndjson-стримом.
- История диалога mini app живёт в памяти сервера отдельно от истории Telegram-бота.
- RAG-настройки (`rag_only`, `study_mode`) и документы читаются/пишутся в общую SQLite-базу,
  поэтому синхронны с ботом.

## Ограничения

- Голосовые сообщения по-прежнему обрабатываются самим ботом в Telegram (Whisper живёт в его процессе).
  Загрузка документов теперь доступна и из mini app, и через бота.
- Пока туннель/сервер выключены, кнопка в Telegram открываться не будет — это нормально
  для локального сценария: запускаете ПК → `run_miniapp.bat` → `tunnel_cloudflare.bat`.
- Для загрузки PDF/DOCX серверу нужны пакеты `pypdf`, `pymupdf`, `python-docx` — они уже есть
  в основном `requirements.txt` проекта и в `requirements-miniapp.txt`.
